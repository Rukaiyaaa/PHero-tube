Function scope: var is function-scoped, it means accessible within the function where it is declared. If declared outside a function, it becomes globally scoped.
Example:
function abc() {
if (true) {
var x = 10;
}
console.log(x);
}
abc();
Here the output will be x=10

Block scope: let is block-scoped, it means only accessible within the block {} in which it is declared.
Example:
function abc() {
if (true) {
let x = 10;
}
console.log(x);
}
abc();
Here no ouput will be shown but give this ReferenceError: x is not defined

Block scope: Like let, const is also block-scoped, it means limited to the block {} where it is declared.

use cases of `null`
1. When anyone want to assign "nothing" or an empty value to a variable, use null.
Example: let user = null;
2. assigning null to a variable to indicate that it has no value.
Example: let user = { name: "Rukaiya" };
user = null;
3. null is commonly used to represent the absence of an object or to signal that a value will be set later.
use cases of `undefined`
1. A variable that is declared but not initialized will be undefined.
Example: let x;
console.log(x);
2. If a function is called without passing all the arguments, the missing ones are undefined.
Example: function greet(name) {
console.log(name);
